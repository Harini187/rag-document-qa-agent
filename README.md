#  Local RAG-based PDF Question Answering System

A lightweight, fully local **Retrieval-Augmented Generation (RAG)** pipeline that lets you ask natural-language questions about the content of any PDF document and get grounded, context-based answers — powered by open-source embedding models and a locally-hosted LLM (via [Ollama](https://ollama.com/)).

No API keys. No cloud costs. Runs entirely on your machine.

---

##  Features

- **PDF ingestion** — extracts raw text from any PDF file
- **Smart chunking** — splits large documents into overlapping, semantically coherent chunks
- **Semantic search** — embeds text using `sentence-transformers` and retrieves the most relevant chunks with `FAISS`
- **Grounded generation** — answers are generated by a local LLM (`gemma3:4b` via Ollama) and are strictly constrained to the retrieved context, minimizing hallucination
- **Fully local & private** — no data leaves your machine

---

##  How It Works (Architecture)

PDF File
   │
   ▼
[1] Text Extraction (pypdf)
   │
   ▼
[2] Chunking (LangChain RecursiveCharacterTextSplitter)
   │
   ▼
[3] Embedding (sentence-transformers: all-MiniLM-L6-v2)
   │
   ▼
[4] Vector Index (FAISS - IndexFlatL2)
   │
   ▼
User Question ──▶ [5] Retrieve Top-K Relevant Chunks
                         │
                         ▼
                  [6] LLM Answer Generation (Ollama - gemma3:4b)
                         │
                         ▼
                    Final Answer

1. **Load PDF** — extract all text page-by-page using `pypdf`.
2. **Chunk text** — split into ~500-character chunks with 100-character overlap to preserve context across boundaries.
3. **Generate embeddings** — convert each chunk into a dense vector using the `all-MiniLM-L6-v2` sentence-transformer model.
4. **Build vector index** — store all embeddings in a FAISS `IndexFlatL2` index for fast similarity search.
5. **Retrieve context** — embed the user's question and fetch the top-k most similar chunks from the index.
6. **Generate answer** — pass the retrieved chunks as context to a local LLM (Gemma 3, 4B) with a strict grounding prompt, ensuring answers come only from the document.

ollama pull gemma3:4b

## Key Design Decisions

- **Chunk size (500) with overlap (100):** balances retrieval precision with preserving context across chunk boundaries.
- **Top-k retrieval (k=3):** keeps the LLM's context focused and relevant, avoiding noise from unrelated sections.
- **Strict grounding prompt:** the LLM is explicitly instructed to answer *only* from the provided context, reducing hallucinated responses.
- **Local-first design:** using Ollama for inference and FAISS for vector search keeps the entire pipeline free of external API dependencies.
